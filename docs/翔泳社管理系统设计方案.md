# 翔泳社游泳馆管理系统设计方案

## 一、系统总体分析
- 系统名称：翔泳社游泳馆管理系统（基于 Spring Boot）
- 技术栈说明：后端 Spring Boot + MyBatis-Plus + Spring Security + JWT；数据库 MySQL；前端 Vue 3 + Element Plus + Axios + ECharts
- 系统目标与应用场景：面向游泳馆会员预约、课程安排、教练管理、收费结算与数据统计的统一管理；适用于中小型体育培训机构
- 用户角色划分：管理员、普通用户（会员/学员）；教练作为受管对象（可选开放教练端，仅查看课表与评价）
- 系统功能模块总览（模块树）
  - 认证与权限：登录/注册、角色权限、JWT
  - 用户中心：个人信息、消费记录、预约记录、评价
  - 课程管理：课程信息、课程分类、课程状态
  - 教练管理：教练档案、资质信息、状态
  - 课表与预约：排课、预约、取消、签到/到课
  - 收费与订单：订单、支付、退款、账单
  - 评价与反馈：课程/教练评价、反馈处理
  - 统计报表：收入统计、预约统计、课程热度
  - 系统设置：参数、公告、字典

## 二、数据库设计（核心）
### 1. 数据库设计思路
- 关系型数据库 MySQL
- 逻辑删除字段：`deleted`（0/1）
- 状态字段：`status`（启用/禁用/草稿等）
- 通用字段规范：`id`、`create_time`、`update_time`、`create_by`、`update_by`

### 2. 数据表设计
1) 表名：`sys_user`（用户表）
- 字段：id/bigint/否/主键；username/varchar/否/登录名；password/varchar/否/加密密码；real_name/varchar/是/姓名；phone/varchar/是/手机号；email/varchar/是/邮箱；avatar/varchar/是/头像；gender/tinyint/是/性别；status/tinyint/否/状态；last_login_time/datetime/是/最后登录；deleted/tinyint/否/逻辑删除；create_time/datetime/否；update_time/datetime/否
- 主键：id
- 索引：username 唯一；phone 普通
- 关系：与角色表多对多

2) 表名：`sys_role`（角色表）
- 字段：id、role_key、role_name、status、deleted、create_time、update_time
- 索引：role_key 唯一
- 关系：与用户多对多

3) 表名：`sys_user_role`（用户角色关联表）
- 字段：id、user_id、role_id、create_time
- 索引：user_id+role_id 唯一
- 关系：用户-角色多对多

4) 表名：`coach_profile`（教练档案表）
- 字段：id、user_id、coach_code、level、specialty、intro、license_no、status、deleted、create_time、update_time
- 索引：coach_code 唯一
- 关系：教练档案与用户一对一

5) 表名：`course`（课程表）
- 字段：id、course_name、course_type、level、duration_min、price/decimal、capacity、cover_url、description、status、deleted、create_time、update_time
- 索引：course_name 普通
- 关系：与排课表一对多

6) 表名：`course_schedule`（排课表）
- 字段：id、course_id、coach_id、start_time、end_time、course_date、location、max_capacity、current_count、status、deleted、create_time、update_time
- 索引：course_id、coach_id、course_date 普通
- 关系：课程/教练一对多，预约表一对多

7) 表名：`reservation`（预约表）
- 字段：id、schedule_id、user_id、reserve_time、status(已预约/已取消/已到课/爽约)、cancel_time、deleted、create_time、update_time
- 索引：user_id、schedule_id 普通
- 关系：用户/排课一对多；订单一对一（可选）

8) 表名：`orders`（订单表）
- 字段：id、order_no、user_id、reservation_id、amount/decimal、pay_status、pay_type、pay_time、refund_status、deleted、create_time、update_time
- 索引：order_no 唯一；user_id 普通
- 关系：与预约一对一

9) 表名：`evaluation`（评价表）
- 字段：id、user_id、course_id、coach_id、reservation_id、score/tinyint、content、status、deleted、create_time、update_time
- 索引：course_id、coach_id 普通
- 关系：评价关联课程/教练/预约

10) 表名：`income_stat`（收入统计表）
- 字段：id、stat_date/date、order_count、total_amount、refund_amount、create_time
- 索引：stat_date 唯一
- 关系：统计表（可由订单聚合生成）

11) 表名：`feedback`（反馈表）
- 字段：id、user_id、type、content、status(未处理/已处理)、reply_content、reply_time、create_time、update_time
- 索引：user_id 普通
- 关系：用户一对多

12) 表名：`notice`（公告表）
- 字段：id、title、content、publish_time、status、create_time、update_time
- 索引：publish_time 普通
- 关系：无

13) 表名：`sys_param`（系统参数表）
- 字段：id、param_key、param_value、remark、status、create_time、update_time
- 索引：param_key 唯一
- 关系：无

## 三、后端设计（Spring Boot）
### 1. 后端整体架构
- 分层结构：Controller / Service / Mapper
- 技术选型：MyBatis-Plus、Spring Security + JWT、Hibernate Validator、统一异常处理、日志审计

### 2. 模块化设计（按功能模块）
1) 认证与权限模块
- 关联表：sys_user、sys_role、sys_user_role
- 业务逻辑：登录/注册、密码加密、JWT 签发与刷新、RBAC
- CRUD：用户注册、新增角色、分配角色、逻辑删除用户
- REST 接口：/auth/login、/auth/register、/auth/logout、/auth/profile
- 权限控制：管理员/普通用户/教练角色权限

2) 用户管理模块
- 表：sys_user
- 逻辑：用户列表、状态启停、重置密码
- 接口：/admin/users（分页查询/新增/修改/删除）
- 权限：管理员可见

3) 教练管理模块
- 表：coach_profile、sys_user
- 逻辑：教练档案维护、资质录入、状态管理
- 接口：/admin/coaches
- 权限：管理员

4) 课程管理模块
- 表：course
- 逻辑：课程创建、上下架、价格/容量调整
- 接口：/admin/courses
- 权限：管理员

5) 排课管理模块
- 表：course_schedule
- 逻辑：课程排班、时间冲突校验、名额管理
- 接口：/admin/schedules
- 权限：管理员

6) 预约管理模块
- 表：reservation、course_schedule
- 逻辑：预约/取消、并发控制、到课状态
- 接口：/user/reservations、/admin/reservations
- 权限：用户仅看自己；管理员全量

7) 订单与收费模块
- 表：orders
- 逻辑：订单生成、支付状态、退款
- 接口：/user/orders、/admin/orders
- 权限：用户/管理员

8) 评价与反馈模块
- 表：evaluation、feedback
- 逻辑：提交评价、评分统计、反馈处理
- 接口：/user/evaluations、/admin/feedbacks
- 权限：用户提交；管理员处理

9) 统计报表模块
- 表：income_stat、orders
- 逻辑：收入统计、预约趋势、课程热度
- 接口：/admin/stats/overview、/admin/stats/income
- 权限：管理员

10) 系统设置模块
- 表：sys_param、notice
- 逻辑：系统参数、公告发布
- 接口：/admin/params、/admin/notices
- 权限：管理员

## 四、后台管理系统 UI 设计（Element Plus）
### 1. 管理端整体设计思路
- 框架结构：登录页 + 左侧菜单 + 顶部栏 + 内容区
- 桌面端优先，兼顾响应式
- 强调效率与可操作性（表格+表单+弹窗为主）

### 2. 菜单与路由规划
- 首页仪表盘
- 用户管理
- 教练管理
- 课程管理
- 排课管理
- 预约管理
- 订单/收费管理
- 评价与反馈
- 统计报表
- 系统设置（公告/参数/权限）

### 3. 页面级 UI 设计（逐页）
1) 登录页
- 布局：居中卡片
- 组件：Form/Input/Button/Checkbox
- 交互：校验提示、登录 loading

2) 仪表盘
- 布局：统计卡片 + 图表区
- 组件：Card/Statistic/Row/Col
- 交互：数据加载 Skeleton、空状态 Empty

3) 用户管理
- 组件：Table/Pagination/Form/Dialog/Popconfirm/Tag
- 交互：条件查询、批量禁用、删除确认

4) 教练管理
- 组件：Table/Form/Dialog/Upload/Tag
- 交互：资质上传、状态切换 Switch

5) 课程管理
- 组件：Table/Form/Dialog/Input/Select
- 交互：上下架、容量与价格校验

6) 排课管理
- 组件：Table/Form/DatePicker/TimePicker
- 交互：时间冲突提示、名额校验

7) 预约管理
- 组件：Table/Drawer/Tag
- 交互：到课状态标记、取消处理

8) 订单/收费管理
- 组件：Table/Descriptions/Tag
- 交互：支付状态展示、退款确认

9) 评价与反馈
- 组件：Table/Rate/Dialog
- 交互：评价筛选、反馈回复

10) 统计报表
- 组件：Card/Select/DatePicker + ECharts
- 交互：维度切换、时间范围筛选

11) 系统设置
- 组件：Form/Table/Dialog
- 交互：参数编辑、公告发布

## 五、前端用户展示页面 UI 设计（Element Plus）
### 1. 用户端设计目标
- 简洁易用、路径清晰
- 支持 PC 与移动端自适应

### 2. 页面规划
- 首页
- 课程列表页
- 教练列表页
- 课程详情页
- 预约表单页
- 预约记录页
- 消费记录页
- 评价页
- 个人中心

### 3. UI 与交互设计
- 首页：Carousel + Card + Tag，突出热门课程
- 列表页：Card/Grid + Pagination
- 详情页：Descriptions + Rate + Tabs（课程介绍/教练介绍/评价）
- 预约页：Form + DatePicker + TimePicker + Select
- 记录页：Table + Tag + Empty
- 个人中心：Form + Upload + Tabs
- 操作反馈：Message/Notification/Popconfirm
- 状态：Skeleton/Empty/Result

## 六、开发实施计划（毕业设计友好）
### 1. 开发阶段划分
1) 需求分析 + 数据库设计
2) 后端基础模块与 CRUD
3) 后台管理系统开发
4) 前端展示页面开发
5) 系统测试与优化

### 2. 测试与验收要点
- 功能完整性：预约、支付、评价、统计闭环
- 数据正确性：订单与预约一致性
- 界面友好性：交互提示与空状态
- 系统稳定性：并发预约与权限访问控制
- 安全性：密码加密、JWT 校验、敏感数据保护
